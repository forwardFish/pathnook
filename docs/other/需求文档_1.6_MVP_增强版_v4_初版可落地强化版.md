# 0. 先给结论：v3 做完后，能不能成为我们讨论的初版？

## 0.1 结论
**v3 做完以后，可以做出“家庭教育方向的 AI 学习成长系统雏形”，但还不够稳，不足以保证它成为我们讨论的“初版产品形态”。**

更准确地说：

### v3 能做到的
- 做出一个明显强于普通报告工具的 MVP
- 让用户第一次感受到：
  - 不是只看错题
  - 而是在看真实 bottleneck
  - 不是只给计划
  - 而是在给更短路径
- 验证首单、二次回来、最小 compare / resume 是否成立

### v3 还不够的地方
如果只按 v3 执行，产品仍然有风险变成：

- 一个更高级的家庭教育诊断工具
- 一个更好的周计划工具
- 一个更像“AI family review”而不是“AI 学习成长系统”的产品

---

## 0.2 为什么 v3 还不够
v3 已经很好，但还缺少 6 个对“初版成立”非常关键的东西：

1. **初版产品定义不够硬**  
   没有把“家庭教育学习成长系统初版”的边界钉死。

2. **家庭教育场景下的核心用户流还不够具体**  
   虽然有 9 个页面和闭环，但没有把“用户第一次用、第二次回来、持续周用”的真实动作链写透。

3. **报告输出结构还不够像“成长系统”，仍有落回“报告工具”的风险**  
   需要更明确规定：
   - 首页怎么说
   - sample-report 怎么证明
   - report 页必须出现什么
   - child detail 必须承担什么角色

4. **输出关卡和高频纠偏还停在概念层偏多**
   需要更明确最小任务库、任务类型、判断方式、结果呈现方式。

5. **缺少“初版成功标准”**
   不只是技术验收，而是：
   - 什么叫这已经是 AI 学习成长系统初版
   - 什么叫仍然只是工具

6. **缺少更明确的 P0 / P1 / P2 开发约束**
   容易做着做着又膨胀。

所以，**如果你的目标只是做出一个可收费 MVP，v3 已经接近够用。**
但如果你的目标是：

> **开发完之后，我就要把它称为“家庭教育方向的 AI 学习成长系统初版”**

那 v3 还应该再强化一次。

# 1. 本强化版的目标

本强化版要解决的不是“更多功能”，而是：

> **确保开发完成之后，Pathnook 至少具备“家庭教育学习成长系统初版”的产品骨架，而不只是一个高级报告工具。**

---

# 2. 重新定义：什么叫“家庭教育学习成长系统初版”

开发完成后，只有满足下面 4 条，Pathnook 才能算“家庭教育学习成长系统初版”。

## 2.1 它必须是一个判断系统
不是只生成报告，而是能稳定回答：

- 当前真实 bottleneck 是什么
- 当前骨架节点是什么
- 哪些不要过度反应
- 这周真正要抓什么

---

## 2.2 它必须是一个路径系统
不是只给 7 天计划，而是能稳定回答：

- 为什么先做这一步
- 这一周在构建什么能力
- 这一步为什么最值
- 这一步结束后下一步是什么

---

## 2.3 它必须是一个训练系统
不是只让用户继续看，而是要让用户在本周至少经历：

- 1～2 个输出关卡
- 1 次真实水平暴露
- 1 次基于输出的纠偏

---

## 2.4 它必须是一个续接系统
不是一次性交付，而是至少支持：

- 第二次回来时看到变化
- 继续当前 focus
- 继续当前 path
- 把当前状态分享给 tutor

---

# 3. 这版强化后，Pathnook 初版的产品定义

## 3.1 一句话定义
> **Pathnook 初版是一个面向家庭教育场景的 AI 学习成长系统。它帮助家长先看清真实学习瓶颈，再找到当前最短能力路径，并通过小型输出关卡与持续纠偏，推动孩子一周一周地更稳地进步。**

---

## 3.2 对外不能再把自己定义成什么
开发完成后，Pathnook 对外**不能只被定义成**：

- AI homework analyzer
- AI study plan tool
- AI tutor handoff report
- AI family review report

这些都太低了。

---

## 3.3 对外应该被定义成什么
开发完成后，对外应该统一被理解成：

- AI learning and growth system
- family learning support first
- evidence-backed diagnosis
- shortest next path
- output checkpoints
- weekly follow-through

---

# 4. 初版必须稳定成立的 5 个核心模块

这 5 个模块做完，Pathnook 才不是普通工具。

## 模块 1：Skeleton Diagnosis
### 必须回答
- 当前卡在哪个骨架节点
- 为什么这个节点比别的更值得先抓
- 这周不要乱抓什么

### 报告页必须出现
- Skeleton focus
- Why this matters now
- Ignore for now
- Leverage point

---

## 模块 2：Shortest Ability Path
### 必须回答
- 为什么先做这一步
- 这一阶段唯一目标是什么
- 下一步是什么

### 报告页必须出现
- Path name
- Path reason
- Stage goal
- Next step hint

---

## 模块 3：Output Gates
### 初版至少支持 4 类
1. Explain
2. Rebuild
3. Transfer
4. Spot the mistake

### 每个 gate 必须包含
- gateTitle
- whyThisGate
- requiredOutput
- passCondition
- commonFailurePattern

---

## 模块 4：High-Frequency Correction
### 初版必须至少支持两种纠偏视角
#### 视角 A：本周纠偏
- most likely repeat pattern
- what to avoid this week
- current correction target

#### 视角 B：跨周纠偏
- stillRepeating
- what improved
- nextCorrectionTarget

---

## 模块 5：Compare + Resume
### 初版必须至少支持
- compare with last report
- current focus
- latest path stage
- next best action
- tutor-ready share

---

# 5. 家庭教育初版的用户流，必须按这个来做

如果这条用户流做不顺，产品再强也不算初版成立。

## 5.1 第一次使用流
1. 用户从首页理解 Pathnook 不是普通 AI 工具
2. 进入 sample-report 看 proof
3. 进入 pricing，理解 one-time / monthly 的差异
4. 上传资料
5. 填最小 intake
6. 获得 diagnosis + shortest path + 7-day plan + output gate
7. 看到支付解锁逻辑
8. 感知产品价值

---

## 5.2 第二次回来流
1. 用户返回 child detail
2. 看到 current focus
3. 看到 latest path stage
4. 看到 last correction target
5. 上传下一轮材料
6. 看到 compare
7. 继续下一轮

---

## 5.3 tutor handoff 流
1. 家长在 report 页生成 share token
2. tutor 打开只读页
3. 看到 diagnosis summary
4. 看到 weekly focus
5. 看到 tutor-ready summary
6. 不需要家长重新解释一遍背景

---

# 6. 报告页必须强化成“初版主战场”

如果你开发后 report 页还是普通“诊断+证据+计划”，那初版不成立。

## 6.1 初版报告页强制结构
报告页必须按下面顺序排：

1. Diagnosis
2. Skeleton Focus
3. Evidence
4. Shortest Ability Path
5. 7-Day Plan
6. Output Gate Highlight
7. Compare（如有）
8. Share

---

## 6.2 Diagnosis 必须包含
- Primary issue
- Secondary issue
- Pattern vs one-off
- What not to overreact to
- This week’s focus
- Skeleton focus
- Why this matters
- Ignore for now
- Most likely repeat pattern

---

## 6.3 Path 区必须包含
- Path name
- Path reason
- Stage goal
- What this week is trying to build
- What comes next

---

## 6.4 Output Gate Highlight 必须单独可见
不要把它埋在某天卡片里看不见。  
至少要有一个独立强调区：

### 标题建议
**This week’s output checkpoint**

### 内容至少包含
- What the child must do
- Why this matters
- What passing looks like
- What usually goes wrong

---

# 7. Child Detail 页必须升级成“成长状态页”

如果 child detail 只是“历史报告列表”，初版不成立。

## 7.1 初版 child detail 强制包含
- Child header
- Current skeleton focus
- Current path stage
- Latest report
- Next output gate
- Last correction target
- Next best action
- Report history

---

## 7.2 不允许只显示
- latest report
- history list

因为这仍然是档案页，不是状态页。

---

# 8. Sample Report 页必须从“说明页”升级成“证明页”

开发完成后，如果 sample-report 还只是概念说明，初版不成立。

## 8.1 Sample Report 强制包含 7 个块
1. Sample Diagnosis
2. Skeleton Focus
3. Sample Evidence
4. Shortest Ability Path
5. Sample 7-Day Plan
6. Example Output Gate
7. Tutor-ready summary

---

## 8.2 Sample Report 的职责
它不是告诉用户“我们会输出这些模块”。

它必须让用户在付费前就觉得：

> **这不是普通 AI 教育工具，它会抓真实 bottleneck，也会给更短路径。**

---

# 9. 首页与 Pricing 页必须强化“系统感”，不是卖功能

## 9.1 首页必须传达的不是“我们有这些功能”
而是：

- We find the real bottleneck
- We show the shortest next path
- We use small output checkpoints
- We help families carry progress forward

---

## 9.2 Pricing 必须卖的是“持续价值”，不是附加特性
所以必须删掉旧表达：

- EN / ES output
- bilingual output
- PDF export 作为主卖点
- tutor workspace 作为 headline feature

### Pricing 真正应该卖的是
- one-time diagnosis = first proof
- monthly = compare + follow-through + continuity
- annual = year-round workflow

# 11. 初版必须补一份“最小输出关卡库”

这是 v3 还不够硬的地方。  
现在在 v4 里直接补上最小库。

## 11.1 Explain Gate
### 适用于
- 概念理解不稳
- 过程顺序不稳
- 容易“会做但说不清”

### 任务格式
Explain in your own words why this problem is solved in this order.

### 通过标准
- 能完整说出步骤顺序
- 不能跳步
- 不依赖提示词

---

## 11.2 Rebuild Gate
### 适用于
- 错误步骤反复出现
- 顺序错乱
- 过程不稳

### 任务格式
Rebuild the correct order from the mistaken work.

### 通过标准
- 能指出哪一步开始偏
- 能重建正确顺序

---

## 11.3 Transfer Gate
### 适用于
- 看懂原题但不会迁移
- 只会记套路，不会用结构

### 任务格式
Solve one similar problem with a changed surface.

### 通过标准
- 能在新题里仍保持同一结构思路

---

## 11.4 Spot-the-Mistake Gate
### 适用于
- 容易粗心
- 过程看似懂，但识别错误弱

### 任务格式
Point out the first step where the work breaks.

### 通过标准
- 能找到第一处关键错误
- 能说明为什么这是关键错误

---

# 12. 初版成功标准：不是“能上线”，而是满足这 8 条

开发完成后，只有下面 8 条成立，才算“家庭教育 AI 学习成长系统初版”成立。

## 12.1 用户第一次看 report 时，不会只觉得“这是一份报告”
而会感觉：
- 它抓到真正 bottleneck 了

## 12.2 用户第一次看 plan 时，不会只觉得“这是周计划”
而会感觉：
- 这是最短能力路径

## 12.3 用户本周至少看到 1 个 Output Gate
而不是只有继续练习

## 12.4 用户能看到本周 correction target
而不是只有建议

## 12.5 第二次回来时能看到 compare
而且 compare 不只是“有进步 / 没进步”

## 12.6 child detail 能承担状态页角色
而不是报告列表页

## 12.7 sample-report 能在付费前证明价值
而不是概念说明页

## 12.8 pricing 不再卖旧特性
而是在卖持续价值

# 15. 给 Codex 的最终执行指令（v4）

```text
Goal:
Ship a launchable first version of Pathnook as a family-learning AI growth system, not just a stronger report tool.

Must-have outcomes:
1. The product must clearly show the real learning bottleneck.
2. The product must show the shortest ability path, not just a weekly plan.
3. The product must include at least 1 visible output checkpoint in the weekly flow.
4. The product must carry correction targets and compare state across weeks.
5. The public site must reflect this positioning.

Strict rules:
- Do not stop at diagnosis + evidence + weekly plan.
- Make skeleton focus visible.
- Make shortest path visible.
- Make at least one output gate visible.
- Make compare / correction target visible.
- Remove all bilingual-output positioning from public pages.
- Do not treat tutor workspace as a current headline feature.
- Make sample-report a proof page, not a placeholder.
- Make child detail a state page, not a history page.
```
