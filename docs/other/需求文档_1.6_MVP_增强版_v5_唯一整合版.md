# 5. 初版必须同时具备的 4 个系统

## 5.1 判断系统
Pathnook 必须能回答：

- 当前真实 bottleneck 是什么
- 当前骨架节点是什么
- 哪些不要过度反应
- 这周真正要抓什么

如果不能稳定回答这些，它就仍然只是一个报告工具。

## 5.2 路径系统
Pathnook 必须能回答：

- 为什么先做这一步
- 这一周在构建什么能力
- 这一步为什么最值
- 这一步结束后下一步是什么

如果只是给 7 天计划，它还不算路径系统。

## 5.3 训练系统
Pathnook 必须让用户在本周至少经历：

- 1～2 个输出关卡
- 1 次真实水平暴露
- 1 次基于输出的纠偏

如果只有“继续复习 / 继续练习”，那它还是普通学习工具。

## 5.4 续接系统
Pathnook 必须支持：

- 第二次回来时看到变化
- 继续当前 focus
- 继续当前 path
- 继续当前 correction target
- 把当前状态分享给 tutor

如果不能承接第二轮，它就还是一次性交付物。

# 6. 初版最核心的 5 个模块

## 模块 1：Skeleton Diagnosis
### 必须回答
- 当前卡在哪个骨架节点
- 为什么这个节点比别的更值得先抓
- 这周不要乱抓什么

### 报告页必须出现
- Skeleton focus
- Why this matters now
- Ignore for now
- Leverage point

## 模块 2：Shortest Ability Path
### 必须回答
- 为什么先做这一步
- 这一阶段唯一目标是什么
- 下一步是什么

### 报告页必须出现
- Path name
- Path reason
- Stage goal
- Next step hint

## 模块 3：Output Gates
### 初版至少支持 4 类
1. Explain  
2. Rebuild  
3. Transfer  
4. Spot the mistake  

### 每个 gate 必须包含
- gateTitle
- whyThisGate
- requiredOutput
- passCondition
- commonFailurePattern

## 模块 4：High-Frequency Correction
### 本周纠偏视角
- most likely repeat pattern
- what to avoid this week
- current correction target

### 跨周纠偏视角
- stillRepeating
- what improved
- nextCorrectionTarget

## 模块 5：Compare + Resume
### 初版必须至少支持
- compare with last report
- current focus
- latest path stage
- next best action
- tutor-ready share

# 7. 家庭教育初版的真实用户流

## 7.1 第一次使用流
1. 用户从首页理解 Pathnook 不是普通 AI 工具  
2. 进入 sample-report 看 proof  
3. 进入 pricing，理解 one-time / monthly 的差异  
4. 上传资料  
5. 填最小 intake  
6. 获得 diagnosis + shortest path + 7-day plan + output gate  
7. 看到支付解锁逻辑  
8. 感知产品价值  

## 7.2 第二次回来流
1. 用户返回 child detail  
2. 看到 current focus  
3. 看到 latest path stage  
4. 看到 last correction target  
5. 上传下一轮材料  
6. 看到 compare  
7. 继续下一轮  

## 7.3 tutor handoff 流
1. 家长在 report 页生成 share token  
2. tutor 打开只读页  
3. 看到 diagnosis summary  
4. 看到 weekly focus  
5. 看到 tutor-ready summary  
6. 不需要家长重新解释一遍背景

# 9. 首页 `/` 的最终要求

## 9.1 首页目标
用户 5 秒内必须知道：

- Pathnook is an AI learning and growth system
- 当前先做 family learning support
- 它不只是报告工具
- 它的价值在于：
  - 看清真实 bottleneck
  - 找到 shortest next path
  - 用 output checkpoints 暴露真实水平
  - 帮助家庭 week by week 地 carry progress forward

## 9.2 首页 Hero 下的 3 个价值点必须改成
- **See the real bottleneck**
- **Find the shortest next path**
- **Turn review into weekly follow-through**

## 9.3 `What families get today` 必须改成 4 个强价值卡片

### Card 1
**See the real learning bottleneck**  
Not just which questions went wrong, but where the learning process is actually breaking down.

### Card 2
**Get the shortest next path**  
See what to work on first, why it comes first, and what can wait.

### Card 3
**Use small output checkpoints**  
Do not stay in passive review. Use small tasks that reveal whether the learning is really holding.

### Card 4
**Carry progress forward week by week**  
Every review should make the next week clearer, not disappear as a one-time report.

## 9.4 Hero 副文案最终建议
**AI-driven clarity, the shortest next path, and steady family follow-through.**

## 9.5 Proof before pay 区必须改成
**See the output before you pay**

- Sample diagnosis
- Sample evidence
- Sample shortest path
- Sample weekly plan
- Sample tutor-ready summary

## 9.6 FAQ 必须新增一问
**How is Pathnook different from a normal AI study tool?**  
Pathnook is not built to generate shortcut answers or generic study plans. It is built to help families see the real bottleneck, choose the most valuable next step, and keep progress moving from week to week.

# 16. 最小 Output Gate 库（初版强制）

## 16.1 Explain Gate
### 任务
Explain in your own words why this problem is solved in this order.

### 通过标准
- 能完整说出步骤顺序
- 不能跳步
- 不依赖提示词

## 16.2 Rebuild Gate
### 任务
Rebuild the correct order from the mistaken work.

### 通过标准
- 能指出哪一步开始偏
- 能重建正确顺序

## 16.3 Transfer Gate
### 任务
Solve one similar problem with a changed surface.

### 通过标准
- 能在新题里仍保持同一结构思路

## 16.4 Spot-the-Mistake Gate
### 任务
Point out the first step where the work breaks.

### 通过标准
- 能找到第一处关键错误
- 能说明为什么这是关键错误

# 19. 初版成功标准：不是“能上线”，而是满足这 8 条

1. 用户第一次看 report 时，不会只觉得“这是一份报告”，而会感觉：它抓到真正 bottleneck 了  
2. 用户第一次看 plan 时，不会只觉得“这是周计划”，而会感觉：这是最短能力路径  
3. 用户本周至少看到 1 个 Output Gate，而不是只有继续练习  
4. 用户能看到本周 correction target，而不是只有建议  
5. 第二次回来时能看到 compare，而且 compare 不只是“有进步 / 没进步”  
6. child detail 能承担状态页角色，而不是报告列表页  
7. sample-report 能在付费前证明价值，而不是概念说明页  
8. pricing 不再卖旧特性，而是在卖持续价值
