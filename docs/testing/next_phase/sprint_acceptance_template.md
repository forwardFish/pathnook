# Next-Phase Sprint Acceptance Template

## Required Sections

- Sprint name and Sprint ID
- Story list covered by the acceptance run
- Requirement IDs audited
- Page tests executed
- Click paths executed
- API/data/AI/ops tests executed
- Governance tests executed
- Pass/fail summary
- Blockers
- Remaining risks
- Evidence links
- Sprint verdict

## Close Rule

A next-phase Sprint may not be described as closed until:
- all Sprint stories are accepted
- this acceptance packet exists
- `docs/requirements/next_phase_traceability_matrix.md` rows owned by the Sprint are green
- `ship`, `document-release`, and `retro` artifacts exist or are ready in the agentsystem workflow

## Minimum Attachments

- traceability audit snapshot
- screenshot pack or browser-evidence manifest
- smoke output or deterministic execution transcript
- blocker register
- release and handoff references where relevant
- final Sprint verdict
