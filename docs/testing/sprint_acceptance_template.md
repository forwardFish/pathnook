# Sprint Acceptance Template

## Required Sections

- Sprint name and ID
- Story list covered by the acceptance run
- Requirement IDs audited
- Page tests executed
- Click paths executed
- API/data/AI tests executed
- Pass/fail summary
- Blockers
- Remaining risks
- Evidence links

## Close Rule

A Sprint may not be described as closed until:
- all stories are accepted
- this acceptance packet exists
- `ship`, `document-release`, and `retro` artifacts exist

## Minimum Attachments

- traceability audit snapshot
- screenshot pack
- test command output or deterministic execution transcript
- blocker register
- final Sprint verdict
