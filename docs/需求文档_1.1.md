# Parent Dashboard 下一阶段开发需求文档（基于当前已完成系统）
**版本**：v2.0-next-phase  
**用途**：基于当前 `full_system_summary.md` 已完成能力，以及 `parent-dashboard-copy-and-pricing-spec.md` 的产品定位、定价与页面规格，定义下一阶段可直接执行的开发需求。  
**目标**：不是重做 MVP，而是把当前“本地完整可运行版本”推进到 **可真实上线的 Beta 版本**。  
**面向对象**：Codex / 开发工程师 / 产品负责人  
**优先市场**：欧美（优先美国英语家长）

---

# 0. 文档目标与结论

## 0.1 这份文档解决什么问题
当前系统已经具备一套本地可运行、可验收、页面和主流程完整的 Parent Dashboard。  
下一步不是“继续搭页面壳子”，而是进入：

1. **真实外部依赖接通**
2. **从 demo/runtime-first 走向 production-beta**
3. **把报告质量、付费转化、周度留存做深**
4. **把产品从“能跑”变成“值得持续付费”**

## 0.2 下一阶段的核心目标
下一阶段只做四件大事：

### 目标 A：上线就绪
把当前本地 COMPLETE 版本，推进到真正可对外开放测试的 Beta。

### 目标 B：付费闭环真实化
把现有 pricing、billing、unlock、portal 体系真正接上 live Creem，而不是只停留在 demo checkout。

### 目标 C：报告质量与信任提升
让报告更像“有判断的学习系统”，而不是“结构化输出工具”。

### 目标 D：订阅价值增强
强化 weekly review、history compare、趋势变化、家长操作提示，让 $39/月 和 $299/年有持续理由。

## 0.3 下一阶段不做什么
这阶段明确不做：
- 原生移动端 App
- tutor 独立账号体系
- 学校/机构多租户 B2B 版本
- 大规模增长实验系统
- 完整 notification center
- 全学科扩张
- 多地区本地化大规模扩张
- 大而全的 AI provider orchestration 平台

# 2. 下一阶段产品目标（面向真实 Beta）

## 2.1 产品定位不变，但产品阶段升级
定位保持不变：

**Upload your child’s recent math work. Get a clear diagnosis and a 7-day action plan.**

但产品阶段从：
- 本地完整可验收 alpha

升级为：
- 可邀请真实家长试用并完成真实付费的 beta

## 2.2 下一阶段要支撑的真实场景
下一阶段产品要真正支撑这几类高价值场景：

### 场景 1：家长第一次试用并完成首单
用户路径：
- 看到 landing
- 看 sample report
- 注册
- 添加 child
- 上传材料
- 支付 $19
- 拿到报告

### 场景 2：家长第二周继续回来上传
用户路径：
- 查看上次报告
- 根据 plan 执行一周
- 再次上传
- 对比变化
- 感知“产品越来越懂孩子”

### 场景 3：家长从单次升级到月付
用户路径：
- 第一次报告觉得有用
- 进入 billing
- 升级 Parent Weekly
- 形成每周复盘节奏

### 场景 4：家长把报告发给 tutor
用户路径：
- 生成 share link
- tutor 打开只读报告
- 家长和 tutor 围绕同一份 evidence 对齐

### 场景 5：低质量 run 被人审
用户路径：
- OCR/模型低置信度
- run 进入 needs_review
- admin 审核
- 家长最终收到更可信的报告

# 3. 下一阶段工作流总图

## 3.1 用户主流程（Beta）
```text
Landing
-> Sign up / Login
-> Add child
-> Upload 5–10 pages
-> Run created
-> Processing / needs review
-> Report unlocked after billing
-> Diagnosis / Evidence / 7-Day Plan
-> Share with tutor
-> Weekly re-upload
-> Compare over time
-> Upgrade to subscription / annual
```

## 3.2 内部运营流程
```text
Upload submitted
-> AI/OCR processing
-> Confidence scoring
-> Auto publish OR needs_review
-> Admin review
-> Publish report
-> Billing unlock / credits update
-> Reminder / retention jobs
-> Deletion / retention cleanup
```

- subscription_created
